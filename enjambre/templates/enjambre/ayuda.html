{% trans 'Ayuda' %}

{% trans '¿Qué es esto?' %}

{% blocktrans %}Una mesa de trabajo multi-agente: varias IAs se sientan como sillas de una misma mesa — CLIs que ya tenés logueados en tu terminal, modelos por API key (guardada en la bóveda cifrada) o la silla gratis que viene de fábrica y anda sin configurar nada. Vos preguntás; las sillas charlan, debaten y, si se lo pedís, fabrican archivos de verdad en la carpeta git de la mesa.{% endblocktrans %}

{% blocktrans %}Si hay Docker configurado, los CLIs corren en contenedores descartables; en el modo portátil (pendrive, sin Docker) corren como procesos de tu sesión y las sillas por API hablan directo con el proveedor.{% endblocktrans %}

{% trans 'Primeros pasos — la escalera' %}

0 {% blocktrans %}Nada que configurar: la silla gratis (Chispa) ya está sentada. Creá una mesa y probá — es el escalón cero de la escalera que ves en la portada.{% endblocktrans %}
1 {% blocktrans %}Entrá a Conexiones: te muestra qué CLIs tienen login detectado en tu máquina (claude, opencode, agy). opencode tiene login gratis si no tenés ninguno.{% endblocktrans %}
2 {% blocktrans %}¿Tenés API keys? Cargalas en Conexiones → API keys: quedan en la bóveda, cifradas con una passphrase que elegís vos.{% endblocktrans %}
3 {% blocktrans %}En Sillas, encendé las que quieras usar (los CLIs vienen apagados). Ahí también editás su personalidad y su modelo.{% endblocktrans %}
4 {% blocktrans %}Creá una mesa, tildá qué sillas se sientan, y preguntá. Para dirigirte a una sola silla, mencionala: @claude revisá este enfoque. ¿Construir de verdad? /armar <pedido> — las sillas editan archivos en la carpeta git de la mesa y cada turno queda commiteado (📁 para explorar y descargar).{% endblocktrans %}

{% trans 'Topologías' %}

{% blocktrans %}Plana — broadcast: todas las sillas ven tu mensaje y responden a la vez. Ideal para charlar, comparar enfoques, debatir.{% endblocktrans %}

{% blocktrans %}Líder — una silla coordina: descompone tu pedido en subtareas (@alias: subtarea), cada trabajador ejecuta la suya y el líder integra el resultado. Ideal para construir.{% endblocktrans %}

{% blocktrans %}Se elige por mesa, en su configuración (⚙). El líder reparte mirando la especialidad de cada silla.{% endblocktrans %}

{% trans 'El toolbelt' %}

{% blocktrans %}Con el toolbelt encendido, las sillas por API key pueden operar esta máquina para diagnosticar y arreglar: las lecturas (reporte del sistema, inspección con una lista blanca de comandos que no cambian nada) corren solas; cualquier cambio te muestra el comando y espera tu OK antes de tocar algo.{% endblocktrans %}

{% blocktrans %}Viene apagado; se enciende en Conexiones. Toda acción — lectura o cambio — queda registrada en la Bitácora. La silla gratis solo conversa: no usa el toolbelt.{% endblocktrans %}

{% trans 'Tus credenciales' %}

{% blocktrans %}CLIs: el login vive en tu terminal; cuando corren en contenedor reciben una copia efímera de la credencial (solo-lectura → tmpfs) que muere al terminar el turno.{% endblocktrans %}

{% blocktrans %}API keys: se guardan en la bóveda, cifradas con una passphrase que elegís vos y que nadie más conoce. Mientras la bóveda está desbloqueada las keys viven descifradas en un archivo de runtime — bloqueala (Conexiones → API keys) cuando termines.{% endblocktrans %}
