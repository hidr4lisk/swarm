{% trans 'Ayuda' %}

{% trans '¿Qué es esto?' %}

{% blocktrans %}Una mesa de trabajo multi-agente: varios CLIs de IA (tus cuentas, ya logueadas en tu terminal) se sientan como sillas de una misma mesa. Vos preguntás; las sillas charlan, debaten y, si se lo pedís, fabrican archivos de verdad en la carpeta git de la mesa. Cada silla corre en un contenedor descartable: nada toca tu sistema.{% endblocktrans %}

{% trans 'Primeros pasos' %}

1 {% blocktrans %}Entrá a Conexiones: te muestra qué CLIs tienen login detectado en tu máquina (claude, opencode, agy).{% endblocktrans %}
2 {% blocktrans %}En Sillas, encendé las de los CLIs que tengas (vienen apagadas). Ahí también editás su personalidad y su modelo.{% endblocktrans %}
3 {% blocktrans %}Creá una mesa, tildá qué sillas se sientan, y preguntá. Para dirigirte a una sola silla, mencionala: @claude revisá este enfoque.{% endblocktrans %}
4 {% blocktrans %}¿Construir de verdad? /armar <pedido> — las sillas editan archivos en la carpeta git de la mesa y cada turno queda commiteado (📁 para explorar y descargar).{% endblocktrans %}

{% trans 'Topologías' %}

{% blocktrans %}Plana — broadcast: todas las sillas ven tu mensaje y responden a la vez. Ideal para charlar, comparar enfoques, debatir.{% endblocktrans %}

{% blocktrans %}Líder — una silla coordina: descompone tu pedido en subtareas (@alias: subtarea), cada trabajador ejecuta la suya y el líder integra el resultado. Ideal para construir.{% endblocktrans %}

{% blocktrans %}Se elige por mesa, en su configuración (⚙). El líder reparte mirando la especialidad de cada silla.{% endblocktrans %}
