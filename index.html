// 01 · UNA MESA, MUCHAS MENTES

Varias IAs, una misma mesa.

Cada silla es una IA con su voz y su tarea. Dejás un pedido; lo discuten en plano o bajo un líder, y el líder cierra — con una respuesta o actuando. Tu navegador, tu máquina, tus cuentas.

swarm — mesa 07
$ mesa nueva "revisá el disco y arreglá el servicio caído"
[líder] reparto: Arq revisa disco, Red mira el servicio…
[arq]   / disco 82% — /var/log se comió 12 GB
[red]   nginx caído: falta reload tras el logrotate
[líder] propongo: limpiar logs + reload nginx  ⧗ espera tu OK

Muchas mesitas, una por tarea. En vivo, en el navegador.

// 02 · DOS FORMAS DE CORRERLA

Instalada, o en un pendrive.

Mismo app, mismo SQLite. Dos envoltorios — elegí el que le sirve a la máquina que tenés enfrente.

INSTALADA

Un comando: python manage.py serve. Sin Docker. (O docker compose up para aislar las sillas de CLI.)

⇄
PORTÁTIL ⟳

Un Python bundleado por SO en un pendrive. PC pelada, sin Python ni Docker → doble-clic y la mesa abre en el navegador.

El kit del técnico: un pendrive que vuelve cualquier PC de cliente una mesa completa.

// 03 · TRES TIPOS DE SILLA

Por CLI, por API key, o local.

Cada silla elige su backend. Mezclalas en la misma mesa.

swarm — conexiones
$ swarm conexiones
cli     claude · opencode · agy  — cuentas ya logueadas, nada se guarda
api     anthropic · openai · openrouter  — keys cifradas en bóveda con passphrase
local   ollama (http)  — corre entero en tu propio hardware

La ruta por API key no necesita ningún binario — es la que hace posible el pendrive.

// 04 · EL FILO

Puede tocar la máquina real — con cuidado.

Prendé el toolbelt (opt-in, apagado por default) y las sillas por API key reciben herramientas que actúan sobre la computadora a la que enchufaste — sin contenedor, sin cápsula. La red de seguridad no es un sandbox; es disciplina:

  • Lo de solo-lectura corre solo. Revisar disco, procesos y archivos: allowlist sin shell, corre sin preguntar.
  • Toda mutación espera tu OK. Un arreglo nunca corre solo: se encola y lo aprobás o rechazás en la Bitácora.
  • Todo queda logueado. Cada lectura y cada cambio va a la bitácora de la mesa, exportable como informe.
⚠ Poder de verdad
Peligroso en manos equivocadas. Habilitalo solo en máquinas que estés autorizado a atender, y leé cada cambio antes de aprobar.

// ARRANCAR

Arriba en un comando.

Cloná y arrancá. Sin Docker, sin cuenta — solo Python 3.12 (o el runtime bundleado del pendrive).

Nativa / portátil (sin Docker)
$ git clone https://github.com/hidr4lisk/swarm.git && cd swarm $ pip install -r requirements-portable.txt $ python manage.py serve

Abre 127.0.0.1:8799. Andá a Conexiones → API keys, cargá una key (se guarda cifrada) y armá una mesa.

Armar el pendrive (Linux + Windows)
$ ./scripts/build_bundle.sh # → dist/swarm-portable/

Copiás dist/swarm-portable/ a un pendrive. En cualquier PC pelada: doble-clic a enjambre.sh (Linux) o Enjambre.bat (Windows).
